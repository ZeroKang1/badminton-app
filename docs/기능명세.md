# 기능 명세서

> 프로젝트: 소꾹 (배드민턴 클럽 모임 관리 시스템)
> 버전: 2.1.0
> 최종 업데이트: 2026-03-02

---

## 1. 시스템 개요

**소꾹**은 배드민턴 클럽 모임을 운영하기 위한 웹 기반 관리 시스템입니다.
회원 관리, 모임 생성, 출석 체크인, 실시간 코트 배정 및 경기 현황 표시를 제공합니다.

### 주요 화면 구성

```
앱 진입 (app.py)
├── 📺 라이브     →  실시간 경기 현황판 (live_board.py)
└── ⚙️ 운영관리  →  관리 대시보드 (admin_page.py)
                     ├── 📊 회원
                     ├── 📅 모임
                     ├── 📝 접수
                     └── 📈 보고
```

## 2. 화면별 기능 명세

---

### 2.1 📺 라이브 (live_board.py)

실시간으로 코트 배정 상태, 대기열, 선수 풀을 표시합니다.
3가지 표시 모드를 지원합니다.

#### 공통 기능

| 기능 | 설명 |
|------|------|
| 모임 선택 | 드롭다운으로 대상 모임 선택 |
| 표시 모드 전환 | 자석 / 리스트 / 전광판 탭으로 전환 |
| 코트 수 표시 | 모임에 설정된 코트 수만큼 표시 |
| 실시간 갱신 | 캐시 TTL 10초, 버튼으로 수동 새로고침 가능 |

---

#### 2.1.1 자석 모드 (🧲 자석)

선수를 색깔 블록(자석)으로 시각화하는 모드입니다.

**코트 영역 (🏟️ 코트)**

| 요소 | 설명 |
|------|------|
| 코트 헤더 | 코트 번호, 경기 경과 시간 표시 |
| Team A / Team B | 각 2명씩 선수 블록 표시 |
| 선수 블록 색상 | 남성: 파란 계열 / 여성: 빨간 계열 |
| 선수 블록 정보 | 이름 + 출생연도 뒤2자리 + 급수 (예: `홍길동85A`) |
| 빈 슬롯 | 점선 박스로 빈 자리 표시 |
| 배정 버튼 (▶ 배정) | 대기열에서 4명 선택하여 빈 코트에 배정 |
| 종료 버튼 (⏹ 종료) | 경기 종료, 점수 입력 후 선수 해제 |

**대기열 영역 (⏳ 대기열)**

| 요소 | 설명 |
|------|------|
| 대기 선수 목록 | `waiting` 상태 선수 칩 목록 |
| 체크박스 선택 | 선수 칩에 체크박스로 다중 선택 |
| 🎮 대기열 버튼 | 선수 풀에서 선택한 선수를 대기열로 이동 |
| ☕ 휴식 버튼 | 선택 선수를 `resting` 상태로 변경 |
| 🚪 퇴장 버튼 | 선택 선수를 `left` 상태로 변경 |

**선수 풀 영역 (👥 선수 풀)**

| 요소 | 설명 |
|------|------|
| 출석 확인 선수 | `checked_in` 상태 선수 목록 |
| 휴식중 선수 | `resting` 상태 선수 목록 |
| 퇴장 선수 | `left` 상태 선수 목록 |
| 체크박스 선택 | 선수 칩에 체크박스로 다중 선택 |
| 복귀 버튼 | 휴식/퇴장 선수를 `checked_in`으로 복귀 |

---

#### 2.1.2 리스트 모드 (📋 리스트)

표 형식으로 코트 현황을 표시하는 모드입니다.

| 요소 | 설명 |
|------|------|
| 코트 현황 테이블 | 코트번호, Team A, Team B, 경과시간, 상태 |
| 배정 버튼 | 대기열 → 빈 코트 배정 |
| 종료 버튼 | 경기 종료 및 결과 저장 |
| 출석 선수 목록 | 좌측: `checked_in` 선수 체크박스 목록 |
| 휴식/퇴장 목록 | 우측: `resting` / `left` 선수 목록 |
| 상태 변경 버튼 | 선택 선수의 상태 변경 (대기열/휴식/퇴장/복귀) |

---

#### 2.1.3 전광판 모드 (📺 전광판)

체육관 TV/모니터에 표시하기 위한 LED 전광판 스타일 화면입니다.

| 요소 | 설명 |
|------|------|
| 배경 | 검정 배경 |
| COURT 섹션 | 코트별 경기중 선수 표시 (초록 네온) |
| TIME 섹션 | 코트별 경과 시간 표시 (빨간 네온, MM:00 형식) |
| NEXT 섹션 | 대기열 다음 순서 선수 표시 (주황 네온) |
| STANDBY 섹션 | 대기 선수 목록 (초록 네온) |
| 폰트 | Orbitron (레트로 LED 스타일) |

### 2.2 ⚙️ 운영관리 (admin_page.py)

#### 2.2.1 📊 회원 탭

**통계 표시**

| 지표 | 설명 |
|------|------|
| 전체 회원 수 | 소프트 삭제 제외한 활성 회원 수 |
| 남성 회원 수 | gender='남' 필터링 |
| 여성 회원 수 | gender='여' 필터링 |

**조회 및 필터**

| 필터 | 설명 |
|------|------|
| 그룹 필터 | 전체 / 특정 그룹명 선택 |
| 성별 필터 | 전체 / 남 / 여 |
| 이름 검색 | 텍스트 입력으로 이름 필터링 |

**편집 모드**

| 모드 | 설명 |
|------|------|
| 조회만 | 읽기 전용 데이터프레임 표시 |
| 1건 수정 | 드롭다운으로 회원 선택 → 개별 수정 폼 |
| N건 일괄수정 | 여러 회원 동시 수정 (데이터 에디터) |

**회원 수정 필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| 이름 | TEXT | 필수 |
| 성별 | SELECT | 남 / 여 |
| 출생연도 | INT | 4자리 연도 |
| 급수 | SELECT | A / B / C / D / 초심 |
| 전화번호 | TEXT | |
| 그룹 | TEXT | 소속 그룹명 |
| 메모 | TEXT | |

**회원 등록 (신규)**

- 상단 확장 패널(expander)에 새 회원 등록 폼 제공
- 위 수정 필드와 동일한 입력 항목

#### 2.2.3 📝 접수 탭

모임별 참가자를 등록하는 출석 체크인 기능입니다.

**워크플로우**

```
1. 모임 선택
2. 모임의 대상 그룹 자동 필터링
3. 그룹 소속 회원 목록 체크박스 표시
4. 참가자 선택 (다중 선택)
5. "💾 참가 명단 최종 확정" 버튼 클릭
6. DB에 participants 레코드 생성/갱신
```

**표시 정보**

| 요소 | 설명 |
|------|------|
| 모임 정보 | 날짜, 장소, 대상 그룹 |
| 회원 목록 | 3열 그리드로 체크박스 표시 |
| 회원 표시 형식 | `이름+출생연도뒤2자리+급수` (예: `홍길동85A`) |
| 현재 등록 수 | 이미 확정된 참가자 수 표시 |

---

## 3. 데이터 흐름

### 참가자 상태 전이도

```
신청완료(registered)
    ↓
확인완료(confirmed)
    ↓
출석확인(checked_in) ←──── 복귀
    ↓           ↘
 대기열(waiting)  휴식(resting)
    ↓               ↘
 경기중(playing)    퇴장(left)
    ↓
 출석확인(checked_in) ← 경기 종료 후 자동 복귀
```

### 코트 배정 흐름

```
선수 풀 (checked_in)
    ↓ [🎮 대기열 버튼]
대기열 (waiting)
    ↓ [▶ 배정 버튼 - 4명 선택]
코트 배정 (playing)
    ↓ [⏹ 종료 버튼 + 점수 입력]
경기 결과 저장 → 선수 풀 복귀 (checked_in)
```
