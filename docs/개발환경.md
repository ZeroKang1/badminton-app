### 테이블 목록

| 테이블 | 설명 | PK 시작값 |
|--------|------|----------|
| `members` | 회원 정보 | 101 |
| `sessions` | 모임 정보 | 101 |
| `participants` | 참가자 명단 (N:M 중간 테이블) | 101 |
| `matches` | 대진표 (경기 배정) | 101 |
| `match_results` | 경기 결과 기록 | 101 |

### 테이블 스키마

#### members (회원)
```sql
id            BIGINT  PK, AUTO (시작: 101)
name          TEXT    NOT NULL
group_name    TEXT    -- '여민소', '민턴크루' 등
gender        TEXT    CHECK IN ('남', '여')
rank          TEXT    -- 'A', 'B', 'C', 'D', '초심'
phone         TEXT
memo          TEXT
created_at    TIMESTAMPTZ  DEFAULT NOW()
deleted_at    TIMESTAMPTZ  -- 소프트 삭제
```

#### sessions (모임)
```sql
id            BIGINT  PK, AUTO (시작: 101)
title         TEXT    NOT NULL
location      TEXT
date          DATE    DEFAULT CURRENT_DATE
start_time    TIME
end_time      TIME
courts_count  INT     DEFAULT 1
memo          TEXT
created_at    TIMESTAMPTZ  DEFAULT NOW()
```

#### participants (참가자)
```sql
id                BIGINT  PK, AUTO (시작: 101)
session_id        BIGINT  FK → sessions(id) ON DELETE CASCADE
member_id         BIGINT  FK → members(id) ON DELETE CASCADE
status            TEXT    DEFAULT '정상'
entry_time        TIME
fixed_partner_id  BIGINT  FK → members(id)  -- 고정 파트너
fixed_game_count  INT     DEFAULT 0
memo              TEXT
```

**status 값 목록:**
- `registered` - 신청완료
- `confirmed` - 확인완료
- `checked_in` - 출석확인
- `waiting` - 대기열
- `playing` - 경기중
- `resting` - 휴식중
- `left` - 퇴장
- `cancelled` - 취소

#### matches (대진표)
```sql
id              BIGINT  PK, AUTO (시작: 101)
session_id      BIGINT  FK → sessions(id) ON DELETE CASCADE
match_order     INT     NOT NULL
court_num       INT
predicted_start TIME
actual_start    TIMESTAMPTZ
actual_end      TIMESTAMPTZ
team_a_p1       BIGINT  FK → members(id)
team_a_p2       BIGINT  FK → members(id)
team_b_p1       BIGINT  FK → members(id)
team_b_p2       BIGINT  FK → members(id)
score_a         INT     DEFAULT 0
score_b         INT     DEFAULT 0
memo            TEXT
```

#### match_results (경기 결과)
```sql
id               BIGINT  PK, AUTO (시작: 101)
session_id       BIGINT  FK → sessions(id)
court_name       TEXT
team_a_players   TEXT[]  -- PostgreSQL 배열
team_b_players   TEXT[]
score_a          INT
score_b          INT
start_time       TEXT
end_time         TEXT
created_at       TIMESTAMPTZ  DEFAULT NOW()
```

### 데이터 캐싱 전략

| 데이터 | TTL | 이유 |
|--------|-----|------|
| 회원 목록 | 60초 | 변경 빈도 낮음 |
| 모임 목록 · 경기 결과 | 30초 | 중간 빈도 |
| 참가자 · 대진표 | 10초 | 실시간 라이브 데이터 |
| 데이터 변경 시 | 즉시 무효화 | `clear_cache()` 호출 |

### 방법 1: Streamlit Community Cloud (권장 - 상시 외부 접속)

> GitHub 저장소와 연동하여 자동 배포. 항상 공개 URL로 접속 가능.

**배포 절차:**

1. **share.streamlit.io** 접속 → GitHub 계정으로 로그인
2. **New app** 클릭
3. 아래와 같이 설정:

   | 항목 | 값 |
   |------|-----|
   | Repository | `ZeroKang1/badminton-app` |
   | Branch | `main` |
   | Main file path | `app.py` |

4. **Advanced settings** → Secrets 탭에서 환경변수 설정 (선택사항)
5. **Deploy** 클릭 → 수 분 내 배포 완료

**결과:** `https://[앱이름].streamlit.app` 형태의 공개 URL 생성

**코드 업데이트 방법:**
```bash
# 로컬에서 수정 후 push하면 자동 재배포
git push origin main
```

---

### 방법 2: GitHub Codespaces (개발/테스트용 외부 접속)

> GitHub에서 클라우드 개발 환경을 생성하고 포트를 공개로 열어 외부 접속.
> `.devcontainer/devcontainer.json` 설정으로 자동 환경 구성.

**Codespace 생성 절차:**

1. **github.com/ZeroKang1/badminton-app** 접속
2. 초록색 **Code** 버튼 클릭
3. **Codespaces** 탭 → **Create codespace on main** 클릭
4. 브라우저 기반 VSCode가 열리고 자동으로 `pip install` 실행됨

**앱 실행:**
```bash
# Codespace 터미널에서
python -m streamlit run app.py
```

**포트 공개 설정:**

1. 하단 **PORTS** 탭 클릭
2. `8501` 포트 우클릭
3. **Port Visibility** → **Public** 선택
4. 포워딩 주소(🌐 아이콘) 복사 → 외부에서 접속 가능

**URL 형태:** `https://[codespace이름]-8501.app.github.dev`

> ⚠️ Codespace를 종료하면 URL이 만료됩니다. 상시 접속이 필요하면 Community Cloud 사용.

### 절차 3: SQL 직접 실행 (SQL Editor)

1. 좌측 메뉴에서 **SQL Editor** (코드 아이콘) 클릭
2. **New query** 클릭
3. SQL 입력 후 **Run** (또는 **Ctrl + Enter**)

**자주 쓰는 SQL 예시:**

```sql
-- 전체 회원 조회 (삭제된 회원 제외)
SELECT id, name, group_name, gender, rank, phone
FROM members
WHERE deleted_at IS NULL
ORDER BY id;

-- 특정 모임의 참가자 조회
SELECT p.id, m.name, m.rank, p.status
FROM participants p
JOIN members m ON p.member_id = m.id
WHERE p.session_id = 101
ORDER BY m.name;

-- 오늘 모임 조회
SELECT * FROM sessions
WHERE date = CURRENT_DATE;

-- 경기중인 선수 조회
SELECT m.name, m.rank, p.status
FROM participants p
JOIN members m ON p.member_id = m.id
WHERE p.status = 'playing';

-- 테이블 초기화 (주의!)
-- DELETE FROM participants WHERE session_id = 101;
```
